# 📋 Task Tracker CLI & GUI PRO

Современная экосистема управления бэклогом проектных задач, распределения исполнителей и жесткого контроля жизненного цикла задач с помощью встроенного конечного автомата (State Machine) [21%]. Проект поддерживает как консольный интерфейс (CLI) [21%], так и полноценное десктопное графическое приложение (GUI) на базе CustomTkinter.

---

## 🛠 Технологический стек и стандарты

* **Язык:** Python 3.12+ (полная совместимость с релизами 3.14+).
* **Среда и зависимости:** Декларативное управление через `pyproject.toml` и менеджер `uv` [21%].
* **База данных:** SQLite (разворачивается автоматически при первом запуске) [21%].
* **ORM:** SQLAlchemy 2.0 (Modern Declarative Syntax: `Mapped`, `mapped_column`) [21%].
* **Отчетность:** Генерация `.xlsx` файлов через `openpyxl` со встроенными нативными формулами Excel (`NOW()`, `AND`) для динамического условного форматирования.
* **Качество кода:** 100% покрытие аннотациями типов, верифицировано через `mypy --strict` [21%].
* **Тестирование:** Набор юнит-тестов на базе `pytest` [21%].

---

## 🚀 Быстрый старт (Инструкция для преподавателя)

Проект оптимизирован под утилиту `uv` [21%]. Настройка окружения и установка зависимостей выполняются автоматически в изолированной песочнице при первом запуске.

### 1. Запуск Графического интерфейса (GUI)
```bash
uv run gui.py
```

### 2. Запуск статического анализа типов (Строгий режим)
```bash
uv run mypy --strict main.py models.py reporter.py gui.py
```
*Команда завершается со статусом `Success: no issues found in 4 source files`, гарантируя абсолютную чистоту типов.*

### 3. Запуск юнит-тестов
```bash
uv run pytest -v test_tracker.py
```
*Запускает 7 комплексных тест-кейсов для верификации матрицы переходов конечного автомата и ООП-логики сортировки приоритетов.*

## 🖥 Консольные команды (CLI-интерфейс)

Интерфейс командной строки спроектирован на базе именованных строгих опций (`--option`).

### 1. Управление пользователями
```bash
uv run main.py user-add --name "Сергей Петров" --role "Разработчик"
```

### 2. Добавление задачи в бэклог
Доступные приоритеты: `low`, `medium`, `high`, `blocker`. При создании задача автоматически получает статус `To Do`.
```bash
uv run main.py task-add --title "Спроектировать архитектуру БД" --priority blocker --deadline "2026-06-30" --description "Использовать паттерн State Machine"
```

### 3. Назначение исполнителя
```bash
uv run main.py task-assign --task-id 1 --user-id 1
```

### 4. Смена статуса (Валидация State Machine)
Цепочка переходов: `To Do` -> `In Progress` -> `Code Review` (синоним: `review`) -> `Testing` -> `Done`.
```bash
uv run main.py task-move --task-id 1 --status inprogress
```

### 5. Просмотр бэклога с умной сортировкой
Выводит бэклог в консоль с автоматическим переносом длинных заголовков по словам (Word Wrap). Задачи сортируются по весу приоритетов (`Blocker` всегда на первом месте).
```bash
uv run main.py backlog --sort priority
```

### 6. Генерация Excel-таймшитов
```bash
uv run main.py timesheet --week 26 --year 2026 --output "weekly_report.xlsx"
```

## 🎨 Интерактивные фичи Графического интерфейса (GUI)

Десктопное приложение `gui.py` расширяет базовый функционал системы до полноценного Enterprise-уровня:

1. **Множественная сортировка бэклога:** Задачи автоматически ранжируются по критичности (от `Blocker` к `Low`), а при одинаковом приоритете выше встает самая срочная задача по дедлайну.
2. **Динамическое назначение сотрудников:** Исполнителя можно выбрать в один клик через выпадающий список (`ComboBox`) как в форме создания задачи, так и прямо внутри строки бэклога в общей таблице.
3. **Фильтрация выполненных задач:** Встроенный чекбокс *«Скрыть завершенные (Done)»* мгновенно убирает с экрана закрытые задачи, позволяя сфокусироваться на активном спринте.
4. **Табличная сетка с автопереносом:** Все столбцы бэклога разграничены четкими визуальными границами. Длинные заголовки задач автоматически переносятся по словам внутри ячеек, сохраняя ровную структуру таблицы.
5. **Интеграция с проводником OS:** При экспорте отчетов за неделю или всей базы данных приложение вызывает нативное диалоговое окно сохранения файлов, позволяя пользователю вручную выбрать папку и задать имя файла.
6. **Цветовая подсветка дедлайнов:** Строки задач в бэклоге окрашиваются в пастельные тона на основе системного времени:
   * 🔴 **Мягкий красный:** Задача просрочена (`Статус != Done` и `Дедлайн < NOW`).
   * 🟡 **Мягкий желтый:** Высокий риск просрочки (до дедлайна осталось менее 24 часов).
   * 🟢 **Мягкий зеленый:** Задача успешно завершена в срок (`Статус == Done` и `Дата закрытия < Дедлайн`).
